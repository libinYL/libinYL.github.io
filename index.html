整体理解

基于maven的目录结构：

配置4种角色：

1）配置文件
2）mapper.xml
3）接口
4）模型

  • config
  • mapper
  • interface
  • model

对照图示理解映射关系：

  1. resource下配置 mybatis-config.xml。此配置文件中包含对jdbc属性的配置（即jdbc.properties），和对众多mapper.xml的引用。
  2. resource下配置jdbc.properties.
  3. resource下配置 com.xxx.mapper目录,在其中添加mapper.xml.
  4. mapper.xml中的namespace要与对应的接口限定名完全相同。

文件模板：

mybatis.config

jdbc.properties

mapper.xml

mysql压缩包安装笔记

发表于 2018-09-12 | 更新于: 2018-09-12

  1. 下载zip并解压。

  2. 根目录添加my.ini文件。

  3. 配置环境变量，加入到path中。

  4. 执行mysqld --initialize，生成数据文件夹data。

  5. 执行mysqld --console即启动mysql服务。（不推荐，可跳过）

  6. 管理员执行mysqld --install，将mysql安装为windows服务。

  7. 管理员执行net start mysql。
  8. 初始密码可查看data文件夹下以.err结尾的文件。
  9. 登录：mysql -uroot -p。

其他问题：

  • 如果提示密码过期：

    ALTER USERroot@localhostIDENTIFIED BY '1234',root@localhostPASSWORD EXPIRE NEVER;

  • 常见url示例：

    jdbc:mysql://localhost/studentsys?serverTimezone=UTC&useSSL=false&allowPublicKeyRetrieval=true;

C++概览

main函数

  • C++要求以int main()开头。
  • int main()<=>int main(void)
  • 避免使用void main()
  • 如果到达main()函数末尾没有遇到返回语句，默认认为以return 0;结尾

头文件名

  • C语言头文件：扩展名为.h
  • C++头文件：1）保留C头文件 2）纯C++头文件没有扩展名，通过namespace std使用 3）有些C头文件被转换为C++头文件，加上前缀c，如math.h->cmath，可能包含新的特性

命名空间

  • 偷懒做法：using naespace std 大型项目会有潜在问题
  • 推荐做法：只使所需名称可用。using std::cout。

语句

语句的特点：以分号结尾。

声明语句 defining declaration 和变量

  • （通常）指出要存储的数据类型和对应数据的名称，即变量，变量=值可修改，如int carrots;。
  • （复杂）引用声明，reference declaration。

赋值语句

  • 赋值语句将值赋给存储单元。carrots=2;

函数

  • 被调用的函数：called function
  • 包含函数调用的函数：calling function

函数原型语句：返回值类型 函数名（参数类型） 如double sqrt(double)，一般定义在头文件中。
