HashMap全解析

发表于 2018-03-16 | 分类于 技·术

什么是Map？

java.util.Map接口定义了一个Map类应该实现的方法。Map维护了Entry接口，每个Entry存储一个键值对，每个Map存储多个Entry。

HashMap

存储形式：

散列表。需要索引来定位位置，达到一定容量后需要扩容。当索引相同时发生冲突。

每个键值对的容器：Node。实现了Map.Entry接口。

创建Node主要需要这个Node的Hash值，要存储的键，值。

如何增加数据？

首先要得到这个值在散列表中的索引。

如何得到一个对象的索引？

常规方法：直接取这个对象的哈希值对数组长度的模。

有没有更好的方法？

存在这样的事实：一个值a，对一个长度为2的n次方的数b取模，等同于这个值a与（b-1）做位与运算。

而位与运算比取模运算要快非常多。

所以更好的方法是，用这个对象的哈希值与数组长度-1做与运算，作为最后的索引。

一旦采取这种方案，在数组初始化、扩容时候，都必须令数组长度为2的n次方。

那么数组初始化的大小应当为大于给定值的第一个大小是2的n次方的值。

何时扩容？

可以等数组满了再扩容。

但是：随着数组逐渐被填满，冲突的概率越来越高。如果等数组元素的数量等于数组长度再扩容，此时很可能已经发生了大量冲突，影响性能。而扩容也有很大开销，所以也不能太早扩容。

因此，变量loadFactor的作用显现出来。每次传入新的元素，都会检查元素个数与临界值（threshold=capacity*loadfactor）的大小。如果大于临界值，则需要进行扩容。

如何扩容？

扩容时需要考虑：是否超过了int最大值？如果扩容会不会超过int最大值？最重要的是，扩容后需要把原数组的数据复制到新的扩容后的数组中。
